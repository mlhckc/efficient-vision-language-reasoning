# Stage 1: Prepare the GQA subset and answer vocabulary

## Purpose

Turn the raw GQA questions into a fixed, reproducible working set for the rest
of the project: a training and a validation split of fixed size, and a closed
answer vocabulary. Fixing these once means every later experiment uses the same
data, so accuracy differences come from the models rather than the inputs.

## Method

The balanced GQA v1.2 questions were used. `1_prepare_gqa.py` downloads the
question archive (about 1.4 GB) into data/gqa/raw if it is absent and extracts
`train_balanced_questions.json` and `val_balanced_questions.json`; the images
are not needed at this stage. Run with:

    python 1_prepare_gqa.py

The steps, all under config.RANDOM_SEED:

1. Load the full balanced train and val questions, keeping question id, image
   id, question text and the short answer.
2. Build the answer vocabulary as the config.TOP_K_ANSWERS most frequent answers
   in the full training set, breaking frequency ties alphabetically so the
   mapping is deterministic.
3. Drop questions whose answer is outside the vocabulary and attach the integer
   label.
4. Sample config.N_TRAIN training and config.N_VAL validation examples with a
   fixed random_state.

Settings come from config.py (TOP_K_ANSWERS = 100, N_TRAIN = 40000,
N_VAL = 8000, RANDOM_SEED = 42).

## Outputs

All written under data/ and results/, which are git-ignored because they are
regenerated by the script:

- data/answer_vocab.json: answer-to-index map, the answer list ordered by label
  index, and the vocabulary size.
- data/train.csv, data/val.csv: columns questionId, imageId, question, answer,
  label.
- results/stage1_prepare_gqa.json: run metadata (git commit, seed, versions,
  GPU) plus the Stage 1 statistics below.
- data/gqa/raw/: the downloaded archive and the two extracted question files.

## Results

All figures are from the run, not estimates.

- Full balanced split: 943,000 train and 132,062 val questions.
- Answer vocabulary: 100 answers. The top answers by frequency are no, yes,
  left, right, man, white, black, bottom, woman, chair, blue, top.
- Coverage: the top 100 answers cover 77.6% of train answers and 77.6% of val
  answers; the remaining questions are dropped as out-of-vocabulary.
- Sampled working set: 40,000 train examples over 27,718 images and 8,000 val
  examples over 4,928 images, all 100 answers present in each split.
- Answer frequencies are skewed: in the full training set no (166,217) and yes
  (165,681) together account for about 35% of answers, with left (49,009) and
  right (48,342) next.
- Determinism: running the script twice produced byte-identical train.csv,
  val.csv and answer_vocab.json (matching md5 sums).

## Decisions and problems

The balanced split was used rather than the full "all" split. It is smaller and
is the standard choice for GQA accuracy work because it reduces question-type
and answer-frequency bias.

The vocabulary was built from the full training answers before sampling, so the
frequency estimates use all 943,000 questions rather than only the 40,000 kept,
which makes the answer set more stable.

The answer distribution is skewed towards yes/no, so a model that always
predicts the majority answer would already reach a notable accuracy. This has to
be kept in mind when reading the baselines in Stage 3; accuracy is reported as
required by the project scope, against the question-only and image-only
baselines that set the bar.

Closed-set classification drops the roughly 22% of questions whose answer is
outside the top 100. This is inherent to the task as defined and will be
revisited when the answer set is scaled to 1000 as a later experiment.

Train and validation come from GQA's separate balanced splits, so there is no
image overlap between them.

The 1.4 GB archive is kept in data/gqa/raw so re-runs skip the download; the
script downloads it automatically only when it is missing.
